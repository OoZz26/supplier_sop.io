1. Cross-Team Communication Protocol

⭐ Why This Section Comes First
Most data quality issues trace back to a communication gap between teams — not to mistakes in the field.

1.1 Who Is Involved

StakeholderResponsibilities
Sales Team • Communicates with stakeholders to take the requirements
• Gathers their requirements: what data they need and why
• Passes those requirements to the Data team
• Does not conduct field visits
Data Team • Receives requirements from Sales
• Defines the exact data fields needed for analysis
• Writes the Feature Requirements Document (FRD)
• Hands the FRD to IT · Validates data quality after collection
IT / Dev Team • Receives the FRD from the Data team
• Builds the data entry form exactly as specified
• Runs testing (UAT with the Data team)
• Deploys the form when sign-off is given
Freelance Field Agency • Receives the deployed form from the system
• Conducts physical visits to the assigned places
• Fills all mandatory fields and captures required images
• Submits one record per visit — no exceptions

1.3 What Sales Must Pass to the Data Team

Minimum required from each stakeholder meeting:

  • Data points requested by the stakeholder — exactly what they want to capture.
  • Any constraints mentioned by the stakeholder (e.g., some info may not be available on-site).
  • Changes vs. the previous campaign (new requirements, removed requirements, etc.).
⚠ RULE — No Direct Sales-to-IT Requests
Sales must not contact IT directly to request form changes. All changes go: Sales → Data team (FRD update) → IT → UAT → Deploy.

3. Pre-Visit Preparation — For the Field Agent

Complete every item below before leaving for the outlet.

Checklist ItemWhat to Do
Log in to the systemConfirm your assigned place list for today is visible and up to date.
Check for existing recordsFor each place, verify no record already exists for this visit period using the CR number. If a record exists — do NOT create a new one update the previous record.
Know the place typeEach place is a type of location: restaurant, café, cold store, supermarket, cafeteria, bakery, etc. You will select this from a dropdown — familiarise yourself with the options before visiting.
Device and camera checkDevice charged · Camera lens clean · Internet access confirmed (or offline form ready).
Know the exception vocabularyReview Section 4. You must use standard note tags for exceptions. You may not invent your own.
📌 DEFINITION — What Is an Outlet?
"Outlet" = the type of place you are visiting. Examples: Restaurant · Café · Supermarket · Cold Store · Cafeteria · Bakery · Grill / Shawarma · Food Truck · Fruits & Vegetables Shop. You select it from a fixed dropdown — never type it as free text.

4. Place Identification — Decision Tree

On arrival, follow this decision tree. One correct action per situation. Do not improvise.

Step 1 — Is the Place physically present?

OutcomeAction
✅ YES — Place foundProceed to Step 2.
❌ NO — Not foundSelect Note: "Place Not Found"
Leave CR Number and Place Name blank.
Submit and move to the next Place.

Step 2 — Does it match the system record?

SituationCorrect Action
✅ MatchesProceed to Section 5.
⚠ Name has changedNote: "Name Changed"
Fill CR if visible. Do NOT change the Place Name field.
⚠ Temporarily closedNote: "Temporarily Closed"
Fill CR and name if visible. Skip all other fields.
⚠ Permanently closed / demolishedNote: "Permanently Closed"

📌 RULE — Standard Notes Only
Standard note tags: "Place Not Found" · "Name Changed" · "Temporarily Closed" · "Permanently Closed"
Select from the dropdown. Free-text may only be added after the standard tag is chosen.

5. Data Entry Field Rules

Only fill these fields after completing Section 4 with a positive outcome. All MANDATORY fields must be completed before submission.

FieldRequiredRules
CR NumberMANDATORY Exactly as shown on the registration plate or certificate.
• Format: XXXXXX-N  (e.g., 30432-1)
• No spaces before or after the dash
• No leading zeros in the suffix: use -1 not -01
• If not visible — leave blank. Do not guess.
Place NameMANDATORY Copy EXACTLY as shown on the registration or official signage.
• Do not correct spelling, capitalisation, or punctuation
• Do not translate between Arabic and English
• The Data team handles all standardisation
Outlet TypeMANDATORY Select from the system dropdown only — never type free text.
• Examples: Restaurant · Café · Cold Store · Supermarket · Cafeteria · Bakery · Grill/Shawarma · Sandwich Shop · Food Truck · Fruits & Vegetables
• If genuinely unclear → select "Other" and add a note
• Do not leave blank
Location / AreaMANDATORY Select from the predefined area list.
• If the area is not listed → select the nearest option and notify your agency supervisor.
Additional NotesOPTIONAL Use only for context beyond the standard note tags.
• One short sentence maximum
• Examples: "Second floor", "Shared premises", "New owner since last visit"

6. Image Capture Guidelines

Every image must be clear and complete. Blurry or incorrectly framed images will be flagged — fixing them requires returning to the place.

Image 1: Registration Image

Full registration certificate or plate — all text (especially CR number and trade name) must be legible.

✅ DO
Full document in frame — no edges cut off
All text legible — hold steady, no blur
Straight-on angle
❌ DON'T
CR number cut off at the edge
Blurry or overexposed
Photo of a photocopy
Image 2: Exterior / Interior Image

Wide-angle shot showing the full place in one frame.

✅ DO
Stand back far enough to include the entire space
Exterior: entrance and signage visible
Interior: full room layout in one shot
❌ DON'T
Close-up of a single item
Half the place cut off
Dark or heavily shadowed image
Image 3: Product / Contents Image

Complete frontal view of the items to document. All items visible in one image.

✅ DO
Full frontal — all items visible
Door or cover fully open before shooting
Sufficient lighting to read labels
❌ DON'T
Angle shot where only part shows
Door partially blocking the view
Shooting through glass without opening
Image 4: Barcode / Label Image

Barcode or product label — all printed information must be readable.

✅ DO
Full barcode and surrounding text in frame
Camera held steady — no motion blur
Close enough to read text without zooming
❌ DON'T
Barcode partially cut off
Glare or reflection covering the barcode
Too far away to read text

7. Duplicate Visit Prevention

Each physical visit must produce exactly one record. The CR number is the unique identifier — the system uses it to detect duplicates.

  • Check before you fill. Confirm no record already exists for this place's CR number in the current visit period.
  • One record per CR number per visit period. If a record already exists, open and edit it — do not create a new one.
  • Never submit a second record to "correct" the first. Use the edit function in the system to update the existing record.
  • IT must enforce this in the system. The form must detect existing records by CR number and offer an edit option — not allow a new submission.

8. Common Errors & How to Avoid Them

ErrorWhy It Matters & Correct Action
CR number with spacesEnter 30432-1, not "30432 - 1". Spaces create unresolvable duplicates.
Leading zeros in CR suffixEnter 30432-1, not "30432-01". Leading zeros make the same place appear twice.
Guessing a CR numberIf not visible, leave blank. An invented CR permanently corrupts the record.
Modifying the place nameCopy exactly as seen. Changed names break matching across visit periods.
Free-text outlet typeAlways use the dropdown. Free-text values cannot be aggregated in analysis.
Submitting a duplicate recordCheck by CR number first. If a record exists, edit it — do not create a new one.
Partial or blurry imagesCheck before leaving. Rejected images require a return visit.
Sales contacting IT directlyAll form changes must go through the Data team FRD process.
